## 3. Фаза 1 — Ядро MVP

### 3.1 Модуль Auth (Аутентификация)

**Backend:**
- [ ] Регистрация по номеру телефона (SMS + OTP)
- [ ] Вход по номеру телефона / email + пароль
- [ ] JWT: access token (15 мин) + refresh token (30 дней), ротация refresh
- [ ] Двухфакторная аутентификация (SMS TOTP)
- [ ] OAuth 2.0: Госуслуги (ЕСИА) — заявка на подключение + интеграция
- [ ] OAuth 2.0: VK ID, Яндекс ID
- [ ] Восстановление пароля (SMS / Email)
- [ ] Блокировка аккаунта при брутфорсе (rate limiting по IP + user_id)
- [ ] Device fingerprinting + sessions management
- [ ] Logout / logout all devices
- [ ] RBAC: роли `client`, `partner`, `expert`, `admin`, `moderator`

**Frontend (Reflex):**
- [ ] Страница входа / регистрации
- [ ] Верификация OTP (SMS)
- [ ] Кнопки OAuth

**Безопасность:**
- [ ] Хэширование паролей (bcrypt/argon2)
- [ ] Защита от timing-атак при сравнении токенов
- [ ] Хранение refresh токенов в httpOnly cookie
- [ ] CSRF-защита

### 3.6 Модуль Search (Поиск)

**Backend:**
- [ ] Полнотекстовый поиск партнёров, услуг, товаров (Elasticsearch)
- [ ] Геопоиск: партнёры в радиусе N км от клиента (Elasticsearch `geo_distance`)
- [ ] Фильтрация: по категории, цене, рейтингу, наличию акции, расстоянию
- [ ] Сортировка: по релевантности, цене, рейтингу, расстоянию
- [ ] Алгоритм ранжирования (рейтинг 40% + цена 25% + расстояние 20% + доступность 10% + акции 5%)
- [ ] Получение ближайших свободных слотов при поиске услуг
- [ ] Кэширование частых запросов (Redis, TTL 5 мин)
- [ ] Автодополнение / подсказки в строке поиска

**Frontend (Reflex):**
- [ ] Страница поиска (услуги / товары / запчасти)
- [ ] Пошаговый фильтр выбора услуги
- [ ] Фильтр каталога товаров
- [ ] Отображение результатов списком и на карте (Яндекс Карты JS API)

## 4. Фаза 2 — Расширенный функционал

### 4.1 Модуль Emergency (Экстренная помощь)

- [ ] Кнопка "Экстренная помощь" с автоопределением геолокации
- [ ] Выбор типа проблемы: эвакуатор, прикурить, топливо, шиномонтаж, застрял, прочее
- [ ] Broadcast-поиск: push всем партнёрам с нужной услугой в радиусе 15 км
- [ ] Расширение радиуса до 30 км при отсутствии ответа
- [ ] WebSocket для получения первого принятого ответа (аналог такси)
- [ ] Таймер для партнёров (60 сек на принятие)
- [ ] Трекинг партнёра в реальном времени (WebSocket + geolocation)
- [ ] Расчёт ETA
- [ ] Канал связи: чат + звонок после принятия заявки
- [ ] Оплата по завершении (фиксированный тариф / расчёт)

---

### 4.2 Модуль Chat (Коммуникации)

- [ ] Чат клиент ↔ партнёр (по заказу)
- [ ] WebSocket сервер для real-time сообщений
- [ ] Хранение истории чата (MongoDB или PostgreSQL JSONB)
- [ ] Фильтр контактных данных в сообщениях (скрытие телефонов, email, ссылок — защита от увода клиентов)
- [ ] Вложения (фото, документы) — без исполняемых файлов
- [ ] Статусы прочтения (delivered / read)
- [ ] Push-уведомление о новом сообщении
- [ ] Чат с экспертом-подборщиком (см. AI Module)
- [ ] Архивирование чатов

---

### 4.3 Модуль Review (Отзывы)

- [ ] Запрос отзыва после закрытия заказа (автоматический через 2 ч после `closed`)
- [ ] Оценка партнёра: 1-5 звёзд + текстовый отзыв
- [ ] Оценка клиента партнёром (для внутреннего рейтинга)
- [ ] Ответ партнёра на отзыв
- [ ] Модерация отзывов (ручная / автоматическая через NLP)
- [ ] Агрегированный рейтинг партнёра (средняя оценка, количество отзывов)
- [ ] Витрина отзывов на странице партнёра
- [ ] Флаги жалоб на отзыв

- [ ] Воронка регистраций партнёров

## 7. Нефункциональные требования

### 7.1 Безопасность (выполняется на всех фазах)

- [ ] WAF + DDoS-защита (Yandex Cloud Shield / Cloudflare)
- [ ] SSL/TLS на всех эндпоинтах
- [ ] Rate limiting: по IP + user_id (FastAPI slowapi / nginx)
- [ ] RBAC: проверка владения ресурсом на каждом эндпоинте
- [ ] Изоляция данных партнёров (row-level security в PostgreSQL)
- [ ] Шифрование ПДн в БД (AES-256)
- [ ] Маскирование телефонов и карт в логах и API
- [ ] Аудит-лог всех действий с ПДн (152-ФЗ)
- [ ] HTTPS-only cookies (httpOnly, Secure, SameSite=Strict)
- [ ] SQL injection защита (ORM parameterized queries)
- [ ] XSS защита (Content-Security-Policy header, escaping в шаблонах)
- [ ] Валидация и санитизация всех входных данных (Pydantic v2)
- [ ] Периодический аудит зависимостей (`pip-audit`, `safety`)

### 7.2 Производительность

- [ ] Целевой p99 латентности API: < 300 мс
- [ ] Кэширование: Redis для поисковых запросов (TTL 5 мин) и статичных справочников
- [ ] Пагинация всех списковых эндпоинтов (cursor-based)
- [ ] Индексы PostgreSQL на полях поиска и внешних ключах
- [ ] Горизонтальное масштабирование: stateless backend (Kubernetes / Yandex Managed K8s)
- [ ] CDN для статики и изображений (Yandex CDN)

### 7.3 Надёжность

- [ ] Managed PostgreSQL с репликацией (Yandex Managed PostgreSQL)
- [ ] Резервное копирование БД: ежедневно, хранение 30 дней
- [ ] Redis Sentinel / Cluster для HA
- [ ] Graceful shutdown FastAPI
- [ ] Идемпотентность критических операций (оплата, создание заказа)
- [ ] Dead-letter queue для Celery задач с ошибками

### 7.4 Соответствие требованиям (Compliance)

- [ ] 152-ФЗ «О персональных данных»: политика, согласие, хранение в РФ
- [ ] 54-ФЗ «О применении ККТ»: кассовые чеки через ЮKassa
- [ ] Политика конфиденциальности + пользовательское соглашение
- [ ] Хранение данных на серверах в РФ (Yandex Cloud, регион `ru-central1`)
